HTML

HTML stands for Hyper Text Markup Language:

  • HyperText means that it uses the HTTP part of the Internet
  • Markup means the code you write is annotated with keywords
  • Language means it can be read by both a human and a computer

Think of HTML (HyperText Markup Language) as the skeleton of the web page. It is used to give content to a web page and instructs web browsers on how to structure that content.

Elements and Structure

HTML STRUCTURE

HTML is organized into a family tree structure. HTML elements can have parents, grandparents, siblings, children, grandchildren, etc. 

 
<body>
    <div>
        <h1>It's div's child and body's grandchild</h1>
        <h2>It's h1's sibling</h2>
    </div>
</body>

Opening Tag

An opening tag denote the beginning of an HTML element. It is made up of the element's symbol wrapped in left and right angle brackets. 

<p>
<div>
<section>

Closing Tag

A closing tag denote the end of an HTML element. The syntax for a closing tag is a left angle bracket < followed by a forward slash / then the element name and a right angle bracket to close >. 

</p>
</div>
</section>

Element Content

The content of an element is the information between the opening and closing tags of an element. This may be text and/or other HTML elements. This is everything a user is able to see on the website (as you are seeing right now). 

<p> Element content goes here </p>

HTML Attribute

Attributes are extra information tied to an HTML element. It adds meaning to the HTML element it's attached to. It lives inside the opening tag of an HTML element. HTML Element

An HTML element is a piece of content in an HTML document and uses the following syntax: opening tag + content + closing tag.

  • <p> the opening tag.
  • Hello World! is the content.
  • </p> the closing tag.

<p>Hello World! </p>

<p>HTML element</p>

<p> Paragraph Element

The <p> element contains and display a block of text. 

<p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>

<h1> to <h6> Heading Element

The <h1> to <h6> HTML elements represent six levels of section headings. <h1> is the highest section level and <h6> is the lowest.

<h1>Breaking News</h1>

<h2>This is the 1st subheading</h2>

<h3>This is the 2nd subheading<h3>

<em>Emphasis Element

It defines emphasis on text and is usually rendered as italic text.

<div> Div Element

The <div> element defines a division or a section in an HTML document. Any sort of content can be put inside the <div> element!

The <br> element defines a line break within a text. The line break element requires only an opening tag and must not have a closing tag.

<head> Head Element

The <head> element contains general information about an HTML page that isn't displayed on the page itself. This information is called metadata and includes things like the title of the HTML document and links to stylesheets.

     <head>
         Metadata goes here
     </head>
 </html>

<img> Image Element

The <img> element defines an image inserted on a webpage. The src attribute contains the image URL and is mandatory. <img> is an empty element meaning it should not have a closing tag.

Tables

Building a table in HTML requires a specific structure:

  • open a <table>
  • add rows with <tr>
  • add regular cells with <td> or heading cells with <th>

This hierarchy is required, and all 3 elements are necessary to build a table.

<caption> Caption Element

It defines the title of a table. 

<table>
  <caption>The Beatles</caption>
</table>

<table> Table Element

It defines a container for tabular data. In HTML, the <table> element has content that is used to represent a two-dimensional table made of rows and columns. 

<table>
  rows and columns will go here
</table>

<tr> Table Row Element

The table row element,<tr>, is used to add rows to a table before adding table data and table headings. 

                            
 <table>
     <tr>
        ...
     </tr>
 </table>

<thead> Table Head Element

The table head element,<thead>, defines the headings of table columns encapsulated in table rows. 

                            
 <table>
      <thead>
          <tr>
             <th>heading1</th>
             <th>heading2</th>
          </tr>
      </thead>
 </table>

<td> Table Data Element

The table data element,<td>, defines a table cell and must be a direct child of a <tr>. 

                            
<table>
    <tr>
        <td>cell one data</td>
        <td>cell two data</td>
    </tr>
</table>

Forms

HTML forms are used to gather input from users for a variety of reasons, from e-commerce checkouts, to contact forms, to email sign-ups, and more.

<form> Form Element

The form element,<form>, just like the way the body element houses every visible part of a webpage, that has been the way the form element serves as a container for collection of inputs from a web user.

<form action=""> </form>

<input> input Element

The input element,<input>, it is a self-closing tag and uses type attribute to determine its purpose and behavior. 

                            
<input type="text">
<input type="email">
<input type="password">
<input type="number">

<textarea> textarea Element

The textarea element,<textarea>, is used when creating a text-box for multi-line input (e.g. a comment section). The element supports the rows and cols attributes which determine the height and width, respectively, of the element.

<textarea rows="10" cols="30" name="comment"></textarea>

<label> label Element

The label element,<label>, element provides identification for a specific <input> based on matching values of the <input>'s id attribute and the <label>'s for attribute. 

                            
<label for="user-name">Your Name</label>
<input type="text" id="user-name" name="user-name">

<select> select Element

The select element,<select>, is used to create a dropdown list and the list of choices is created using the option element <option>. 

                            
<select>
    <option>Option 1</option>
    <option>Option 2</option>
    <option>Option 3</option>
</select>

Semantic HTML

The semantic HTML refers to the HTML elements that provide meaning to an HTML page rather than just presentation. It makes HTML more comprehensible by better defining the different sections and layout of web pages.

<header> header Element

The header element,<header>, should be used to contain introductory content for a page or article. It might contain things such as the title, a tagline, authorship information, etcetera. 

                            
<header>
     <h1>HTML Reference</h1>
</header

<nav> nav Element

The nav element,<nav>, defines a container for navigation links. 

                            
<nav>
   <a href="/">Home</a>
   <a href="/about">About</a>
   <a href="/contact">Contact</a>
</nav>

<main> main Element

The main element,<main>,represents the primary content of a page. 

                             
<main>
    <h1>My blog post</h1>
    <p>Lorem ipsum dolor sit amet. </p>
    <p>etc.</p>
</main>

<section> section Element

The section element,<section>,is the most nonspecific of the semantic elements and represents a thematically-related section of a page. 

                             
<section>
    <h2>Section title</h2>
    <p>Lorem ipsum dolor sit amet.</p>
</section>

<figure> figure Element

The figure element,<figure>, is used to contain a useful image, illustration, diagram, code snippet, et cetera, that helps to explain the content in the text. And the figcaption element defines caption of the figure 

                             
<figure>
    <img src="/images/html-reference-logo.png" alt="HTML Reference logo">
    <figcaption>The HTML Reference logo</figcaption>
</figure>

<footer> footer Element

The footer element,<footer>, should be used to contain content about the creator or creation of a page or article. It might contain things such as publication date, authorship information, links to related content, etcetera. 

                             
<footer>
    <p>Site by So and So.</p>
    <p>Copyright 2022</p>
</footer>
